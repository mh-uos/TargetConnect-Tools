# Employability & Enterprise Hub - RSS Widgets

This repository hosts custom-built, responsive frontend widgets designed to display live Employability & Enterprise events and vacancies. The HTML files are designed to be embedded seamlessly into University SharePoint and Canvas pages via iframes.

## Project Overview
Instead of relying on restrictive third-party widget providers, these custom widgets pull live data directly from TargetConnect's public RSS feeds. They parse the raw XML, extract relevant hidden fields (such as "Closing Date" or "Employer"), dynamically sort the data, and render it into accessible, mobile-responsive UI cards and lists.

## How It Works
* **Hosting:** Hosted securely on GitHub Pages as static files.
* **Languages Used:** Vanilla HTML5, CSS3, and JavaScript. Zero external UI libraries or dependencies.
* **Data Processing:** Client-side JavaScript fetches the TargetConnect XML feed, bypassing browser CORS restrictions via a standard proxy (`corsproxy.io`). It processes the data in the user's browser without needing a backend server.

## Security & Privacy Statement
This project is built with strict adherence to data security and IT compliance:
* **No PII or Student Data:** The code does not collect, track, process, or store any personal user data.
* **No Backend Server:** This is a 100% static, client-side application. There are no databases to compromise.
* **Read-Only Data:** The widgets only parse publicly accessible, anonymized RSS feeds generated by TargetConnect. There are no API keys, passwords, or access tokens within the code.
* **Access Control:** Write-access to this repository is restricted to a single authorized user and is secured by Two-Factor Authentication (2FA).

## Repository Files
* `index.html` - The **Events Feed Widget**. Displays events in a responsive grid layout. Extracts the "Closest Future Date" to the top of the card.
* `vacancies.html` - The **Vacancies Feed Widget**. Displays job opportunities in a stacked list layout. Sorts dynamically by closest closing date and extracts Employer, Opportunity Type, and Closing Date into colored visual tags.

## Maintenance
To update the TargetConnect feed source, simply edit the `rssUrl` variable located at the top of the `<script>` block inside the respective HTML file. The `cb=` (cache-buster) timestamp ensures that users always receive the most up-to-date feed upon page load.
